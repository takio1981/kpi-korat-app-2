สิ่งที่ต้องมี (Prerequisites)
Node.js และ npm: สำหรับใช้ build โปรเจกต์ Frontend และ API
Docker และ Docker Compose: สำหรับใช้รันแอปพลิเคชันในรูปแบบ Container
ขั้นตอนการใช้งาน
ขั้นตอนที่ 1: ตั้งค่าโปรเจกต์ครั้งแรก (ทำครั้งเดียว)
ติดตั้ง Dependencies:

เปิด Terminal ไปที่โฟลเดอร์ frontend แล้วรันคำสั่ง npm install
เปิด Terminal ไปที่โฟลเดอร์ api แล้วรันคำสั่ง npm install
วัตถุประสงค์: เพื่อติดตั้งไลบรารีและเครื่องมือทั้งหมดที่จำเป็นสำหรับ-การ build โปรเจกต์
สร้างไฟล์ Environment (.env):

ในโฟลเดอร์หลักของโปรเจกต์ (ที่เดียวกับ docker-compose.yml) ให้คุณคัดลอกไฟล์ .env.example แล้วเปลี่ยนชื่อเป็น .env
เปิดไฟล์ .env ขึ้นมาแล้วแก้ไขข้อมูลการเชื่อมต่อฐานข้อมูล (Database) ของคุณให้ถูกต้อง เช่น
DB_HOST=your_database_host
DB_USER=your_username
DB_PASSWORD=your_password
DB_NAME=your_database_name
DB_PORT=3306
วัตถุประสงค์: ไฟล์นี้จะถูกอ่านโดย Docker Compose เพื่อส่งค่า Environment Variables ที่จำเป็น (เช่น ข้อมูล DB) ไปให้ API service ตอนที่มันทำงาน
ขั้นตอนที่ 2: Build โปรเจกต์
เมื่อใดก็ตามที่คุณมีการแก้ไขโค้ดใน frontend หรือ api และต้องการนำไปรันบน Docker ให้ทำตามขั้นตอนนี้

รัน Build Script:
เปิด Command Prompt หรือ PowerShell ที่โฟลเดอร์หลักของโปรเจกต์
รันคำสั่ง:
.\build.bat
สิ่งที่เกิดขึ้น: สคริปต์นี้จะทำงานอัตโนมัติ 3 อย่าง:
ลบโฟลเดอร์ dist ของเก่าใน frontend และ api เพื่อให้แน่ใจว่าไม่มีไฟล์เก่าปะปน
สั่ง npm run build ใน frontend เพื่อสร้างโค้ดเวอร์ชัน production
สั่ง npm run build ใน api เพื่อเตรียมไฟล์สำหรับรัน
เมื่อสคริปต์ทำงานเสร็จ คุณจะได้โฟลเดอร์ dist ที่เป็นเวอร์ชันล่าสุดใน frontend และ api
ขั้นตอนที่ 3: รันแอปพลิเคชันด้วย Docker
สั่ง Docker Compose Build และ Run:
เปิด Command Prompt หรือ PowerShell ที่โฟลเดอร์หลักของโปรเจกต์ (ที่เดียวกับที่รัน .build.bat)
รันคำสั่ง:
docker-compose up --build
สิ่งที่เกิดขึ้น:
Docker Compose จะอ่านไฟล์ docker-compose.yml
--build จะสั่งให้ Docker สร้าง image ของ frontend และ api ขึ้นมาใหม่โดยอิงตาม Dockerfile ของแต่ละตัว (ซึ่งจะทำการ COPY โฟลเดอร์ dist ที่คุณเพิ่งสร้างในขั้นตอนที่ 2 เข้าไปใน image)
หลังจากสร้าง imageเสร็จ Docker จะทำการ start container ของ frontend และ api ขึ้นมา
ขั้นตอนที่ 4: การเข้าใช้งาน
เข้าถึงหน้าเว็บ (Frontend): เปิด Web Browser แล้วไปที่ http://localhost:8808
API: ตัว API จะทำงานอยู่ที่ http://localhost:8809 แต่โดยปกติแล้วเราจะไม่ได้เรียกใช้งานโดยตรง แต่หน้าเว็บ (Frontend) จะเรียกผ่าน Nginx Proxy ที่ตั้งค่าไว้ให้แล้ว
สรุป Workflow การทำงาน (เมื่อมีการแก้ไขโค้ด)
แก้ไขโค้ดใน frontend หรือ api
รัน .\build.bat เพื่อสร้าง build ใหม่
รัน docker-compose up --build เพื่อสร้าง image และรัน container ใหม่
Refresh หน้าเว็บ http://localhost:8808 เพื่อดูการเปลี่ยนแปลง
การปิดแอปพลิเคชัน
ในหน้าต่าง Terminal ที่คุณรัน docker-compose up อยู่ ให้กด Ctrl + C
หากต้องการหยุดและลบ container ทั้งหมด ให้รันคำสั่ง docker-compose down

คุณพูดถูกครับ ในขั้นตอนที่เรารันคำสั่ง เราไม่ได้พิมพ์คำสั่ง copy โดยตรง แต่ การคัดลอก (COPY) นั้นเป็นคำสั่งที่ซ่อนอยู่ และเป็นหัวใจสำคัญของไฟล์ Dockerfile ที่เราสร้างกันไปครับ

ให้มองว่า docker-compose up --build คือคำสั่งให้ "เริ่มสร้าง" โปรเจกต์ตาม "พิมพ์เขียว" ซึ่งพิมพ์เขียวนั้นก็คือไฟล์ Dockerfile นั่นเองครับ

คำสั่ง COPY อยู่ตรงไหน?
คำสั่ง COPY อยู่ในไฟล์ Dockerfile ของแต่ละ service ครับ

1. ในไฟล์ frontend/Dockerfile:

บรรทัดนี้คือส่วนที่ทำหน้าที่คัดลอกโฟลเดอร์ frontend/dist/browser ที่สร้างโดย build.bat เข้าไปใน Image ของ Nginx ครับ

# ... (คำสั่งอื่นๆ) ...

# 2. Copy the pre-built Angular application
# The user builds the app on the host, and we copy the output (``dist/browser``)
# into the ``/kpikorat`` subdirectory to match the base-href and Nginx config.
COPY dist/browser /usr/share/nginx/html/kpikorat

# ... (คำสั่งอื่นๆ) ...
COPY dist/browser: คือการบอกให้ Docker มองหาโฟลเดอร์ dist/browser บนเครื่องของคุณ (ที่เพิ่งสร้างเสร็จจาก build.bat)
/usr/share/nginx/html/kpikorat: คือการบอกว่าให้คัดลอกไฟล์ทั้งหมดไปวางไว้ที่ path นี้ ข้างใน Docker Image ที่กำลังจะสร้าง
2. ในไฟล์ api/Dockerfile:

เช่นเดียวกันกับ Frontend บรรทัดนี้ทำหน้าที่คัดลอกโฟลเดอร์ api/dist เข้าไปใน Image ของ API ครับ

# ... (คำสั่งอื่นๆ) ...

# 3. Copy pre-built application code and PM2 configuration
# The user builds the app on the host, and we copy the output.
COPY dist/ ./dist/
COPY ecosystem.config.js ./

# ... (คำสั่งอื่นๆ) ...
COPY dist/ ./dist/: คือการคัดลอกโฟลเดอร์ dist จากเครื่องของคุณ ไปยังโฟลเดอร์ dist ที่อยู่ใน Work Directory (/app) ของ Image
สรุปให้เห็นภาพ
คุณรัน .\build.bat -> ผลลัพธ์คือมีโฟลเดอร์ dist เกิดขึ้นบนเครื่องคอมพิวเตอร์ของคุณ
คุณรัน docker-compose up --build -> Docker เริ่มทำงาน และอ่านพิมพ์เขียว (Dockerfile)
เมื่อ Docker อ่านมาถึงบรรทัด COPY dist/... ในพิมพ์เขียว -> มันจะทำการคัดลอกโฟลเดอร์ dist จากเครื่องของคุณเข้าไปเก็บไว้ใน Image ที่กำลังสร้างขึ้นมา
เมื่อ Image สร้างเสร็จ Docker จึงนำ Image นั้นไป khởi chạyเป็น Container ที่พร้อมทำงาน
